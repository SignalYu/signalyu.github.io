Virtual DOM#

  1. A Virtual DOM is a lightweight, in-memory representation of the real DOM.
  2. Each Node in the Virtual DOM is represented as a plain JavaScript object with properties describing the element, its attributes, and its children.
  3. The Virtual DOM will be updated whenever an application’s state changes. It minimizes real DOM updates by batching changes and applying them through an optimized process.

Data Binding#

The v-bind directive in Vue provides one-way data binding, allows dynamically binding an element’s attribute or property to a JavaScript expression. The shorthand for v-bind is a single colon (:).

SHOW CODE 
<!-- Full Syntax -->
<div v-bind:attribute="expression"></div>
<!-- Shorthand Syntax -->
<div :attribute="expression"></div>

<img v-bind="image" />

const app = Vue.createApp({
  data() {
    return {
      image: {
        src: "image.jpg",
        alt: "A beautiful image",
        class: {
          cat: isVisible,
          image: !isVisible,
        }
      },
      isVisible: true
    };
  }
});

app.mount('#app');

The v-model directive in Vue is used to create two-way data binding between input form elements and the component’s data. Modifiers such as trim and number can be applied to enhance its functionality. When the input data changes, the corresponding component data updates automatically.

SHOW CODE 
<input v-model.number="age" />
<input v-model.trim="username" />

<div id="app">
  <input v-model="name" placeholder="Enter your name" />
  <p>Hello, {{ name }}!</p>
</div>

<script>
const app = Vue.createApp({
  data() {
    return {
      name: ""
    };
  }
});

app.mount('#app');
</script>

Iterate over Collections#

The v-for directive in Vue is used to iterate over collections such as arrays or objects in JavaScript. It is crucial that the value assigned to the key attribute is both unique and type of string or number. The key attribute allows Vue to efficently update the DOM by tracking each item, ensuring predictable behavior when the underlying data changes.

SHOW CODE 
<li v-for="(item, index) in items" :key="index">
  {{ index + 1 }}. {{ item }}
</li>
data() {
  return {
    items: ["Apple", "Banana", "Cherry"]
  };
}

<li v-for="(value, key) in user" :key="key">
  {{ key }}: {{ value }}
</li>

data() {
  return {
    user: { name: "John", age: 30, location: "USA" }
  };
}

A common approach to ensure uniqueness is to assign a UUID value to the key attribute.

Adding Event Listeners to Elements#

The v-on directive in Vue is used to add event listeners to elements. It listens for DOM events and executes the specified JavaScript function. The shorthand syntax for v-on is @. Vue also provides modifiers like .stop and .prevent to simplify handling common event-related tasks.

SHOW CODE 
<!-- Full syntax -->
<button v-on:click="handleClick">Click Me</button>
<!-- Shorthand syntax -->
<button @click="handleClick">Click Me</button>

<button @click="handleClick('Hello')">Click Me</button>
<script>
export default {
  methods: {
    handleClick(message) {
      console.log(message); // Outputs: Hello
    }
  }
}
</script>

<button @click="handleClick($event)">Click Me</button>
<script>
export default {
  methods: {
    handleClick(event) {
      console.log(event.target); // Logs the clicked element
    }
  }
}
</script>

<!-- Prevent default behavior -->
<form @submit.prevent="handleSubmit">
  <button type="submit">Submit</button>
</form>
<!--  keyboard-specific events -->
<input @keydown.enter="submitForm" placeholder="Press Enter to Submit" />

Event Propagation: The DOM follows a tree-like structure. Events propagate in three phases: starting from the top-most element (document) during the capture phase, moving down to the target element, where the target phase occurs and any associated JavaScript code is executed (if defined), and then propagating back up during the bubbling phase. By default, event listeners in JavaScript and Vue operate in the bubbling phase. However, switching to the capture phase is possible by using the .capture modifier.

SHOW CODE 
<button @click.capture="handleClick">Click Me</button>

Conditional Rendering#

Conditional rendering in Vue is typically achieved using the v-if, v-else-if, v-else, and v-show directives. The v-if directive controls the visibility of an element by conditionally mounting or unmounting it from the DOM. The v-else-if and v-else directives are used alongside v-if to create more complex conditional redering logic. In contrast, the v-show directive is useful in scenarios where elements need to be toggled frequently. It works by toggling the CSS display property, meaning the element remains in the DOM even when the condition is false, but its visibility is controlled via CSS.

SHOW CODE 
<div id="app">
  <p v-if="isVisible">This is visible!</p>
  <button @click="toggleVisibility">Toggle Visibility</button>
</div>

<div id="app">
  <p v-if="status === 'success'">Success!</p>
  <p v-else-if="status === 'loading'">Loading...</p>
  <p v-else>Something went wrong.</p>
  <button @click="changeStatus">Change Status</button>
</div>

<div id="app">
  <p v-show="isVisible">This is visible!</p>
  <button @click="toggleVisibility">Toggle Visibility</button>
</div>

v-html#

The v-html directive is used to bind raw HTML to an element, enabling dynamic rendering of content. It is particularly useful for rendering rich text, such as HTML elements like <strong or <a>. However, using this directive can introduce security risks, as it allows untrusted content or malicoious code (including JavaScript within <script> tags) to be injected into the DOM, making the application vulnerable to XSS (Cross-Site Scripting) attacks. Therefore, it is essential to sanitize any dynamic HTML content before binding it with v-html to prevent such vulnerabilities.

SHOW CODE 
<template>
  <div>
    <p v-html="content"></p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      content: '<a href="https://example.com">Click here</a> to visit our site.'
    };
  }
}
</script>

v-text#

The v-text directive is used to display plain text by binding a value to an element’s text content. Unlike v-html, it escapes any HTML tags and renders the content as plain text. It is useful for text placeholders, as the predefined placeholder text will be replaced once the component has finished loading.

SHOW CODE 
<template>
  <div>
    <p v-text="message"></p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: 'Hello, <strong>Vue</strong>!'
    };
  }
}
</script>
SHOW OUTPUT 
<p>Hello, <strong>Vue</strong>!</p>
SHOW CODE 
<template>
  <div>
    <p v-text="message">Placeholder Text</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: 'Hello, Vue!'
    };
  }
}
</script>

The above code will render the application displaying Placeholder Text and will eventually replace it with Hello, Vue! received from message.

v-once#

The v-once directive is used to render an element or a component only once. After the initial render, Vue will skip re-rendering it, even if its reactive data changes. This directive is useful for displaying static content, such as headers or footers, that do not change after the initial render.

SHOW CODE 
<div id="app">
  <h1 v-once>This header is static and won't re-render</h1>
  <p>{{ message }}</p>
  <button @click="changeMessage">Change Message</button>
</div>

<script>
  new Vue({
    el: '#app',
    data: {
      message: 'Hello, Vue!'
    },
    methods: {
      changeMessage() {
        this.message = 'Message changed!';
      }
    }
  });
</script>

v-memo#

The v-memo directive, introduced in Vue 3, optimizes the rendering of DOM elements by memoizing their output based on specified dependencies. When the dependencies remain unchanged, the element will not be re-rendered, improving performance by avoiding unnecessary updates.

SHOW CODE 
<div id="app">
  <p v-memo="[message]">{{ message }}</p>
  <button @click="changeMessage">Change Message</button>
</div>

<script>
  new Vue({
    el: '#app',
    data: {
      message: 'Hello, Vue!'
    },
    methods: {
      changeMessage() {
        this.message = 'Message changed!';
      }
    }
  });
</script>

In the above code, <p> tag will not re-rendered unless the message changes.

Registering a Global Component#

In Vue, registering a global component allows it to be used across all instances without needing to import it every time. To register a global component, follow these steps:

  1. Define the component – Create the component as a standalone file.
  2. Register the component – In the main.js or main.ts file, use app.component to globally register the component.
  3. Use the component – After registration, the component can be used in any Vue instance within the application without needing to import it again.
SHOW CODE 
<!-- Define the Component -->
<template>
  <div>
    <h1>Hello from MyComponent!</h1>
  </div>
</template>

<script>
export default {
  name: 'MyComponent'
}
</script>

<!-- Register the component globally -->
import { createApp } from 'vue'
import App from './App.vue'
import MyComponent from './components/MyComponent.vue'

const app = createApp(App)

// Register MyComponent globally
app.component('MyComponent', MyComponent)

app.mount('#app')

<!-- Use the global component in any template -->
<template>
  <div>
    <MyComponent />
  </div>
</template>

<script>
export default {
  name: 'AnotherComponent'
}
</script>
